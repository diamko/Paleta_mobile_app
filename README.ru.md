Paleta Mobile — Android-приложение для [Paleta](https://github.com/diamko/Paleta), инструмента генерации, редактирования, сохранения и экспорта цветовых палитр.
Загрузите изображение для извлечения доминирующих цветов, генерируйте случайные палитры, настраивайте их через цветовое колесо и пипетку, экспортируйте в форматы для дизайнеров и разработчиков.

Приложение предназначено для дизайнеров, мобильных разработчиков и всех, кто работает с цветом и хочет быстро получать HEX-коды прямо с телефона.

## Зачем Paleta Mobile

### Цель

Перенести полный рабочий процесс Paleta на Android — генерация, редактирование, сохранение и экспорт цветовых палитр с телефона.

### Какую проблему решает

- Десктопные инструменты для работы с палитрами неудобны, когда нужно захватить цвета с фото на ходу.
- Перенос палитр между устройствами требует лишних шагов.
- Большинство мобильных утилит для цвета не поддерживают профессиональные форматы экспорта (GPL, ASE, ACO).

### Чем отличается

- Два режима генерации: по изображению (с пипеткой на изображении) и случайная палитра.
- Встроенное цветовое колесо, RGB-слайдеры и цветовые гармонии.
- Экспорт в 6 форматов: JSON, GPL, ASE, CSV, PNG, ACO.
- Гостевой режим — работает без регистрации для быстрых задач.
- Синхронизация с бэкендом Paleta для хранения палитр и недавних загрузок.

## Ключевые функции

- Генерация палитры из изображения с извлечением доминирующих цветов (KMeans на сервере).
- Случайная генерация палитр с режимами гармоний (комплементарная, триадная, аналоговая, расщепленно-комплементарная).
- Пипетка на изображении: перетаскивайте маркеры по загруженному изображению для выбора точных цветов пикселей.
- Цветовое колесо с регулятором яркости для точной настройки цвета.
- RGB-слайдеры для тонкого управления.
- Экспорт палитр: `JSON`, `GPL`, `ASE`, `CSV`, `PNG`, `ACO`.
- Аутентификация (регистрация, вход, восстановление пароля через email).
- Личная библиотека палитр с поиском, фильтрами и сортировкой.
- Недавние загрузки изображений (за 7 дней) для быстрого повторного использования.
- Гостевой режим — полная генерация, редактирование и экспорт без входа.
- Поддержка темной/светлой/системной темы.
- Двуязычный интерфейс: русский и английский.

## Технологии

- `Kotlin 2.0+`
- `Jetpack Compose` (Material 3)
- Архитектура `Single-module MVVM`
- `Retrofit 2` + `OkHttp` для сетевых запросов
- `Kotlin Serialization` для JSON
- `DataStore Preferences` для локальных настроек
- `Kotlin Coroutines` для асинхронных операций
- `Navigation Compose` для навигации между экранами
- `Android SDK 35` (minSdk 26)

## Как это работает

1. Пользователь загружает изображение или генерирует случайную палитру.
2. Для изображений: бэкенд запускает кластеризацию KMeans и возвращает доминирующие цвета.
3. Пользователь редактирует цвета через цветовое колесо, RGB-слайдеры или пипетку на изображении.
4. Палитру можно экспортировать в файл или сохранить в аккаунт.
5. Сохраненные палитры доступны в разделе "Мои палитры" с поиском, фильтрами и сортировкой.

## Установка и настройка

### Требования

- `git`
- `JDK 17` (требуется AGP)
- `Android SDK` (API level 35)
- Android Studio (рекомендуется) или Gradle CLI

### 1) Клонирование репозитория

```bash
git clone https://github.com/diamko/Paleta_mobile_app.git
cd Paleta_mobile_app
```

### 2) Установка JDK 17 (если не установлена)

Ubuntu/Debian:

```bash
sudo apt install openjdk-17-jdk
```

macOS (Homebrew):

```bash
brew install openjdk@17
```

### 3) Настройка пути к Android SDK

Создайте файл `local.properties` в корне проекта:

```properties
sdk.dir=/path/to/your/Android/Sdk
```

### 4) Принятие лицензий SDK (если не сделано)

```bash
$ANDROID_HOME/cmdline-tools/latest/bin/sdkmanager --licenses
```

## Запуск проекта

### Вариант A: Android Studio

1. Откройте папку проекта в Android Studio.
2. Дождитесь завершения Gradle Sync.
3. Выберите конфигурацию `app` и нажмите **Run**.

### Вариант B: Командная строка

```bash
chmod +x gradlew
./gradlew assembleDebug
```

Debug APK будет находиться в `app/build/outputs/apk/debug/app-debug.apk`.

## Конфигурация

### Режим репозитория

По умолчанию приложение использует `FakeRepository` (офлайн, без бэкенда).

Для подключения к реальному бэкенду Paleta измените `app/build.gradle.kts`:

```kotlin
buildConfigField("String", "REPOSITORY_MODE", "\"remote\"")
buildConfigField("String", "API_BASE_URL", "\"http://your-server:5000/\"")
```

### Тема

Приложение поддерживает светлую, темную и системную темы. Переключение доступно в Настройках.

### Язык

Поддерживаются русский и английский. Переключение доступно в Настройках — приложение перезапустится для применения новой локали.

## Как пользоваться

### Гостевой режим (без регистрации)

Доступно:

- генерация палитр из изображений и случайная генерация,
- редактирование цветов через цветовое колесо, слайдеры и пипетку,
- экспорт палитр во все поддерживаемые форматы.

### Режим авторизованного пользователя

Дополнительно доступно:

- сохранение палитр в личную библиотеку,
- переименование и удаление палитр,
- поиск, фильтрация и сортировка в разделе "Мои палитры",
- повторное использование недавних загрузок.
