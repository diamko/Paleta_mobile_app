## Зачем Paleta Mobile

### Цель

Перенести полный рабочий процесс Paleta на Android — генерация, редактирование, сохранение и экспорт цветовых палитр прямо с телефона.

### Какую проблему решает

- Десктопные инструменты неудобны, когда нужно захватить цвета с фото на ходу.
- Перенос палитр между устройствами требует лишних шагов.
- Большинство мобильных утилит не поддерживают профессиональные форматы экспорта (GPL, ASE, ACO).

### Чем отличается

- **Два режима генерации**: по изображению (с пипеткой) и случайная палитра.
- **Встроенное цветовое колесо** с регулятором яркости, вводом HEX, кнопкой Copy HEX и 9 режимами цветовых гармоний.
- **Экспорт в 6 форматов**: JSON, GPL, ASE, CSV, PNG, ACO.
- **Гостевой режим** — полная генерация, редактирование и экспорт без регистрации.
- **Оффлайн-режим** — палитры кэшируются локально и синхронизируются при восстановлении связи.

## Ключевые функции

| Категория | Возможности |
| --- | --- |
| **Генерация** | Палитра из изображения (KMeans на сервере), случайная палитра с режимами гармоний |
| **Редактирование** | HSV цветовое колесо с регулятором яркости, пипетка на изображении с лупой, ввод HEX, Copy HEX, HSV-градиент между первым и последним цветом |
| **Гармонии** | Аналоговая, комплементарная, триадная, квадратная, расщепленно-комплементарная, монохроматическая, последовательная, тетрадная, произвольная |
| **Экспорт** | JSON, GPL, ASE, CSV, PNG, ACO |
| **Библиотека** | Личная библиотека палитр с поиском, фильтрацией по кол-ву цветов и сортировкой (имя, дата, кол-во) |
| **Авторизация** | Регистрация, вход, восстановление пароля по email, редактирование профиля, смена пароля |
| **Оффлайн** | Кэш в Room с фоновой синхронизацией через WorkManager |
| **Интерфейс** | Тёмная / светлая / системная тема, русский и английский язык |

## Архитектура

Проект следует **Clean Architecture** в рамках одного модуля с тремя слоями:

```text
core/       → DI-контейнер, сетевой слой, алгоритмы палитр, интерфейсы хранилищ
data/       → Retrofit API, DTO, Room БД, реализации репозиториев, воркер синхронизации
domain/     → Доменные модели, интерфейсы репозиториев, use-кейсы
ui/         → Compose-экраны, ViewModel, тема, навигация
```

Поток данных: **UI → ViewModel → UseCase → Repository → API / Room**

## Как это работает

1. Пользователь загружает изображение или генерирует случайную палитру.
2. Для изображений: бэкенд запускает кластеризацию KMeans и возвращает доминирующие цвета.
3. Пользователь редактирует цвета через цветовое колесо, HEX-ввод или пипетку на изображении.
4. Цвета можно гармонизировать через 9 режимов (аналоговый, триадный, произвольный и др.) или создать HSV-градиент между первым и последним цветом.
5. Палитру можно экспортировать в файл или сохранить в аккаунт.
6. Сохранённые палитры кэшируются локально (Room) и синхронизируются с сервером в фоне.

## Установка и настройка

### Требования

- `git`
- `JDK 17` (требуется Android Gradle Plugin)
- `Android SDK` (API level 35)
- Android Studio (рекомендуется) или Gradle CLI

### 1) Клонирование репозитория

```bash
git clone https://github.com/diamko/Paleta_mobile_app.git
cd Paleta_mobile_app
```

### 2) Установка JDK 17 (если не установлена)

Ubuntu / Debian:

```bash
sudo apt install openjdk-17-jdk
```

macOS (Homebrew):

```bash
brew install openjdk@17
```

### 3) Настройка пути к Android SDK

Создайте файл `local.properties` в корне проекта:

```properties
sdk.dir=/path/to/your/Android/Sdk
```

### 4) Принятие лицензий SDK (если не сделано)

```bash
$ANDROID_HOME/cmdline-tools/latest/bin/sdkmanager --licenses
```

## Запуск проекта

### Вариант A: Android Studio

1. Откройте папку проекта в Android Studio.
2. Дождитесь завершения Gradle Sync.
3. Выберите конфигурацию `app` и нажмите **Run**.

### Вариант B: Командная строка

```bash
chmod +x gradlew
JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64 ./gradlew assembleDebug
```

Debug APK будет в `app/build/outputs/apk/debug/app-debug.apk`.

## Конфигурация

### Подключение к бэкенду

По умолчанию приложение подключается к удалённому бэкенду Paleta. URL и режим репозитория настраиваются в `app/build.gradle.kts`:

```kotlin
buildConfigField("String", "REPOSITORY_MODE", "\"remote\"")
buildConfigField("String", "API_BASE_URL", "\"https://diamko.ru/\"")
```

Для разработки без бэкенда переключитесь на встроенный фейковый репозиторий:

```kotlin
buildConfigField("String", "REPOSITORY_MODE", "\"fake\"")
```

### Тема

Приложение поддерживает светлую, тёмную и системную темы. Переключение — в **Настройках**.

### Язык

Поддерживаются русский и английский. Переключение — в **Настройках**, приложение перезапустится для применения локали.

## Как пользоваться

### Гостевой режим (без регистрации)

Доступно:

- генерация палитр из изображений и случайная генерация
- редактирование цветов через цветовое колесо, HEX-ввод и пипетку
- применение цветовых гармоний
- экспорт палитр во все поддерживаемые форматы

### Режим авторизованного пользователя

Дополнительно доступно:

- сохранение палитр в личную библиотеку
- переименование и удаление палитр
- поиск, фильтрация и сортировка в разделе «Мои палитры»
- повторное использование недавних загрузок (за 7 дней)
- смена пароля и редактирование профиля
